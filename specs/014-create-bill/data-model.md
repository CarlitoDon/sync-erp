# Data Model: Create Bill

## Entities

### Invoice (Existing - No Changes Needed)

The `Invoice` model in Prisma already supports manual bills because `orderId` is optional:

```prisma
model Invoice {
  id            String        @id @default(uuid())
  companyId     String
  orderId       String?       // ✅ Already optional - can be null for manual bills
  partnerId     String        // Required - supplier for bills
  type          InvoiceType   // INVOICE or BILL
  status        InvoiceStatus @default(DRAFT)
  invoiceNumber String?
  dueDate       DateTime
  amount        Decimal       @db.Decimal(15, 2) // Total = subtotal + taxAmount
  subtotal      Decimal       @default(0) @db.Decimal(15, 2)
  taxAmount     Decimal       @default(0) @db.Decimal(15, 2)
  taxRate       Decimal       @default(0) @db.Decimal(5, 2)
  balance       Decimal       @db.Decimal(15, 2) // Starts equal to amount
  createdAt     DateTime      @default(now())
  updatedAt     DateTime      @updatedAt

  company  Company   @relation(...)
  order    Order?    @relation(...)  // Optional
  partner  Partner   @relation(...)
  payments Payment[]
}
```

### Partner (Reference)

Bills must reference a Partner with `type = SUPPLIER`:

```prisma
model Partner {
  id        String      @id @default(uuid())
  companyId String
  type      PartnerType // Must be SUPPLIER for bills
  name      String
  ...
}

enum PartnerType {
  CUSTOMER
  SUPPLIER
}
```

## Validation Rules

| Field     | Rule                                        |
| --------- | ------------------------------------------- |
| partnerId | Required, must exist, must be SUPPLIER type |
| subtotal  | Required, must be > 0                       |
| taxRate   | Optional, 0-100 (percentage), defaults to 0 |
| dueDate   | Optional, defaults to 30 days from creation |
| notes     | Optional, free text                         |

## Calculated Fields

| Field     | Formula                      |
| --------- | ---------------------------- |
| taxAmount | `subtotal × (taxRate / 100)` |
| amount    | `subtotal + taxAmount`       |
| balance   | `amount` (at creation)       |

## State Transitions

```
DRAFT → POSTED (via post action)
DRAFT → VOID (via void action)
POSTED → PAID (when payments = amount)
POSTED → VOID (not allowed if any payments)
```

## Document Numbering

- Pattern: `BILL-{sequence}`
- Example: `BILL-000023`
- Generated by `DocumentNumberService.generate(companyId, 'BILL')`
